Delivery process

From first call to launch – a predictable, five-step model.

My process is intentionally simple. You always know what happens next, what I need from you, and when you can expect a concrete result.

Discovery → Architecture → Design & content → Build → Launch
Alignment
We start by agreeing on goals, constraints, and scope. No endless workshops – a focused discussion.
Execution
Then I move through architecture, content and implementation in fast, visible iterations.
01 – Discovery & goals

Context, stakes and constraints.

45–60 minute call. We map your current situation, target audience, offers and technical constraints (hosting, existing brand, content sources).

Goals & metrics Target users Existing assets You + me
02 – Architecture

Page map & layout matrix.

I design the structure: which pages exist, how they connect, and what each page is responsible for. Deliverable is a clear architecture document and static layout previews.

Sitemap Page roles Layout matrix Architecture-first
03 – Design & content

Offer-driven copy and visual direction.

Using my Adobe design, content and marketing training, I define the visual language and draft the key messages, offers, headlines and CTAs for every page.

Visual direction Headline system Offer hierarchy Adobe-backed workflow
04 – Build

Static implementation & testing.

I build the site as clean HTML/CSS, tuned for performance, accessibility basics and responsive behaviour. We review on a real staging URL, not in screenshots.

Responsive checks SEO basics Staging link Ready to deploy
05 – Launch & optimisation

From “live” to learning.

Once live, we connect analytics and define a simple improvement loop: which actions matter, what we measure and which small layout or copy changes we try first.

Analytics A/B ideas Campaign hooks Optional ongoing support
